## What this project is

A personal health tracking system that records daily walks (distance in miles), body weight, blood pressure, and blood sugar. Data is visualised as SVG charts using gnuplot, with trend lines fitted via linear regression.

## Key workflow

The main daily workflow is:

```bash
bash makegraph.sh        # regenerate walk.csv, recent.csv, plot the SVG, then commit & push
```

Which internally runs:
1. `perl sort.pl > walk.csv` — scans `walk*.txt` files for the current year, extracts distances, accumulates a running total
2. `tail -n 7 weight.csv > recent.csv` — last 7 weight entries for short-term trend
3. `tail -n 7 walk.csv > recentWalk.csv` — last 7 walk entries for short-term trend
4. `gnuplot ... newFormatWlkWt.gnu` — produces `file.svg`
5. `./commit.sh` — git add, commit, push

Other scripts:
```bash
bash weightFit.sh        # fits a linear model to weight.csv, writes weightFit.csv
bash bp.sh               # generates blood pressure chart via plot_bp.sh
python3 split_sugar.py   # splits sugar.csv into fasting.csv and postprandial.csv
```

## Data formats

| File | Format | Notes |
|------|--------|-------|
| `walk*.txt` | Free text, one line like `Around Scissett - 2.49 Miles` | Filename encodes date: `walkDDMMYYYY.txt` |
| `walk.csv` | `YYYYMMDD\tdistance\tcumulative` (tab-separated) | Generated by `sort.pl` |
| `weight.csv` | `YYYYMMDD\tkg` | Hand-maintained |
| `recent.csv` | Last 7 rows of `weight.csv` | Generated by `makegraph.sh` |
| `recentWalk.csv` | Last 7 rows of `walk.csv` | Generated by `makegraph.sh` |
| `bp.csv` | Blood pressure readings | Plotted by `bp.sh` |
| `sugar.csv` | Blood sugar; col 3 starts with `f` (fasting) or `p` (postprandial) | Split by `split_sugar.py` |

## Core Perl module

`Days.pm` is a local module (loaded via `use lib cwd`) providing date utilities built on `Date::Calc`:
- `numberOfDays($filename)` — parses `walkDDMMYYYY.txt` filename, returns days since 2017-01-01 (used for sorting)
- `dateFromDays($startDate, $n)` — adds n days to a date string `DDMMYYYY`
- `isLeapYear($year)`

Scripts that use `Days.pm` must be run from the repo root so `cwd` resolves correctly.

## gnuplot scripts

`newFormatWlkWt.gnu` is the primary chart script. It:
- Plots weight (left y-axis, kg) and cumulative walk distance (right y-axis, miles) against date
- Fits linear trends to both full history and recent data
- Expects variables passed via `-e`: `datafile` (weight CSV) and `recent` (recent weight CSV)
- Outputs `file.svg`

The x-range and y-ranges are hardcoded for the current year (2026) — update these when the year changes.
